# 待入账收入永远不释放

**发现日期**: 2026-04-23
**状态**: 已修复（2026-04-24）
**影响**: 所有已完成订单的收入永久卡在"待入账"状态，无法入账到 Shopee 钱包

## 现象

大量订单完成（delivered）后，收入一直显示为"待入账"，预计入账日期已过但仍未释放。

## 根因

### Bug 1（核心）：回填函数传错了 `credited_at`

**文件**: `backend/apps/api-gateway/app/api/routes/shopee.py:3226`

```python
credited_at=order.delivered_at or current_tick,
```

对已完成订单，`order.delivered_at` 永远非空，所以 `credited_at` 始终等于送达时间本身。

而释放判断在 `shopee.py:2959-2960`：

```python
release_at = order.delivered_at + timedelta(days=3)
if credited_at < release_at:
    return False  # 不释放
```

结果：`delivered_at < delivered_at + 3天` → 永远 True → **回填永远不会释放任何收入**。

**修复方向**: 应改为 `credited_at=current_tick`，用当前游戏时间与释放阈值比较。

### Bug 2：游戏时间与真实时间混用

**文件**: `shopee.py:2959`

`timedelta(days=3)` 是 3 个真实日历天（259200 秒），而设计意图是 3 个游戏日。

当前游戏设定：365 游戏日 = 7 真实天，因此：
- 3 游戏日 ≈ 4973 秒 ≈ **83 分钟**
- 3 真实天 = 259200 秒 ≈ **3 天**

等待时间被放大了约 52 倍。

**修复方向**: 应使用 `timedelta(seconds=3 * REAL_SECONDS_PER_GAME_DAY)` 替代 `timedelta(days=3)`。

## 影响链路

```
订单送达 → 创建待入账记录（正确）
→ 回填尝试释放 → credited_at 用了 delivered_at 而非 current_tick → 永远判定"未到期"
→ 即使修复 Bug1，timedelta(days=3) 还要等 3 真实天而非 3 游戏日
→ tick worker 不触发回填，只有用户刷页面时才跑一次（但仍因 Bug1 失败）
```
