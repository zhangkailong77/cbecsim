# 游戏时钟与本地时间存在 8 小时偏差

**发现时间**：2026-04-23
**状态**：已定位根因，迁移方案待执行

## 现象

日志中 `current_game_tick` 与实际本地时间相差 8 小时：

```
2026-04-23 17:19:47 - [order-auto-sim] ... current_game_tick=2026-04-23 09:19:47
```

实际本地时间 17:19:47 (CST)，`current_game_tick` 显示 09:19:47 (UTC)。

## 根因

- MySQL Docker 容器时区为 UTC（`SYSTEM`，容器 OS 为 UTC）
- `created_at` 使用 `server_default=func.now()`，`NOW()` 返回 UTC
- 代码中全部使用 `datetime.utcnow()`，也是 UTC
- 所有计算正确（UTC - UTC），但显示给人看的时间是 UTC 而非本地时间

## 影响

- 功能无影响，游戏时钟计算正确
- 日志中的时间戳与本地时间不一致，排查时需要心算 +8
- 前端展示的游戏时间也是 UTC 基准
