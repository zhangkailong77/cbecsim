- 已明确限时抢购流量漏斗：`活动曝光机会 -> 商品浏览(view) -> 商品点击(click) -> 下单(order)`；本期不模拟提醒设置，提醒设置数固定为 `0`。
- 已定义浏览/点击事件表 `shopee_flash_sale_traffic_events` 的字段、索引、约束与统计口径；数据页商品浏览量读取 `view` 事件数，商品点击数读取 `click` 事件数，CTR 按 `click/view` 计算；首版基础浏览概率 `base_view_prob` 固定为 `0.30`，浏览概率上限为 `0.75`。

### 2.14 库存预占与发货 409 修复（2026-04-21）
- 排查并修复"仓库库存够但前端提示库存不足"问题：
  - 根因：`variant.stock`（展示层）与 `inventory_lots.quantity_available`（lot 层）长期不同步，下单时用展示层判断有货，但 lot 层实际为 0，导致订单全部标为 `backorder`，取消时 `stock_release=0`，lot 层库存永久丢失。
  - 修复下单逻辑：`variant_available_stock` 改为直接查 `inventory_lots.quantity_available`，以 lot 层为唯一真实来源；移除 `shortfall` 追加兜底逻辑。
  - 修复发货 409：`consume_reserved_inventory_lots` 按 product 维度 FIFO 消耗，多订单共享 reserved pool 导致先发货订单消耗后续订单的预占；新增 `consume_available_inventory_lots`，reserved 不足时从 `quantity_available` 补足差额。
  - 涉及文件：`backend/apps/api-gateway/app/services/shopee_order_simulator.py`、`backend/apps/api-gateway/app/services/inventory_lot_sync.py`、`backend/apps/api-gateway/app/api/routes/shopee.py`
