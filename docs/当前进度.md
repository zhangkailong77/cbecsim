# 当前开发进度（实时）

最后更新：2026-03-18

## 1. 总体阶段
- 当前系统主链路已覆盖：`Step 01 开局 -> Step 02 选品采购 -> Step 03 物流清关 -> Step 04 入仓 -> Step 05 店铺运营（Shopee）`。
- 当前开发重心已从“页面复刻与表单打通”进入“运营模拟（买家池/订单生成）MVP”阶段。

## 2. 已完成模块（开发侧）
### 2.1 工作台（Setup）
- 左侧菜单 + 右侧内容区架构稳定。
- 支持按菜单切换子视图，并与 URL 子路由同步。
- 统一时间中心与游戏日历组件已接入，支持实时游戏时刻显示与手动查看月份。

### 2.2 Shopee 店铺运营（Step 05）
- 新增商品主流程（第二步详情页）大部分结构已复刻并中文化。
- 核心板块：基础信息、规格、商品描述、销售信息、物流、其他。
- 图片/视频上传、类目选择、规格字段动态渲染、变体/批发价/物流配置等交互已打通到主要数据结构。
- “我的产品”列表已接入发布数据，支持父商品与变体展开查看、批量勾选、底部批量操作条。

### 2.3 管理员买家池总览（yzcube）
- 新增管理员入口菜单：`买家池总览`（仅 `super_admin + yzcube` 可见）。
- 新增买家画像首版（10人）并落库。
- 总览页支持：
  - 对局筛选（run_id）
  - 实时指标（买家总数、当前时段活跃估算、预估每小时订单）
  - Top3 活跃买家
  - 买家画像表格
- 右侧统一时间中心已支持按筛选对局同步显示（对局/玩家/游戏日/游戏时刻）。
- 新增“模拟一小时订单”手动触发按钮（管理员侧）。
- 触发后展示本次模拟摘要：激活买家数、候选商品数、生成订单数、跳过原因。

### 2.4 上架后自动模拟订单（P1.5）
- 已新增订单模拟服务层：复用买家池画像与概率参数，支持按对局产单。
- 已新增管理员触发接口：`POST /game/admin/runs/{run_id}/orders/simulate`。
- 已新增玩家触发接口：`POST /shopee/runs/{run_id}/orders/simulate`（用于调试）。
- 已新增订单模拟日志表：`shopee_order_generation_logs`（含表注释/字段注释维护）。
- Shopee 订单列表接口已扩展：返回最近1小时模拟新增订单数与最近模拟时间。
- 玩家端“我的订单”页已接入中文提示：
  - 顶部提示“最近1小时新增 X 单（买家池系统模拟）”
  - 订单行新增“系统模拟”标签
  - Tab 与关键字段文案中文化（待出货/运输中/已完成等）。
- 已修复手动模拟“激活买家长期为0”的关键问题：
  - 手动触发默认模拟时刻会按上次日志自动 `+1小时` 推进（不再重复同一小时）
  - 买家激活与下单采样改为每次触发独立随机抽样，避免固定哈希导致重复结果。
- 管理员“本次模拟摘要”已升级为可追溯明细：
  - 店铺上下文（run_id / user_id / 玩家 / 市场 / 状态）
  - 买家逐条决策流水（激活判定、候选商品、选中商品、SKU、价格、概率、最终原因）
  - 生成订单详情（订单号、SKU、数量、单价、实付）
- 买家池已扩容至 50 人（`BYR001~BYR050`），昵称统一英文名。
- 50 人均使用多峰活跃时段模型（早/午/晚多个活跃峰值），不再是单一活跃时段。
- 已修复“模拟订单不选择变体”问题：
  - 订单候选筛选改为支持“变体库存总和”参与判断（不仅看父商品库存）
  - 变体商品下单时按买家画像选择最匹配变体并扣减对应变体库存
  - 父商品库存与变体库存总和保持同步更新。
- 已修复“订单列表图片仍显示主图”问题：模拟订单明细图片优先写入变体图，缺失时才回退主图。
- 已修复 Shopee 订单页金额单位显示问题：订单金额前缀统一为 `RM`（马来站展示）。
- Shopee“我的订单”操作列已对齐真实结构：支持三项纵向操作（查看物流详情 / 查看详情 / 打印面单）。
- 已修复模拟订单物流渠道来源：下单时按商品上架勾选的物流方式（标准快递/标准大件/快捷快递）写入订单渠道，不再固定写死。
