# 23-折扣对下单概率的影响设计

## 1. 概述

本文档描述单品折扣活动对订单模拟器中 `price_score` 计算逻辑的影响，以及 `price_sensitivity` 字段的正式接入。

---

## 2. 现有 price_score 计算（改造前）

```python
target_price = 30 + float(buyer.purchase_power or 0.5) * 300
price = max(1, int(listing.price or 0))
price_gap = abs(price - target_price) / max(target_price, 1)
price_score = clamp(1 - price_gap, 0.0, 1.0)
```

**问题：**
- 使用 `abs`，导致低于目标价和高于目标价受到同等惩罚，逻辑不合理
- `price_sensitivity` 字段存在于 `SimBuyerProfile` 但未被使用

---

## 3. 改造后 price_score 计算

### 3.1 核心公式

```python
target_price = 30 + float(buyer.purchase_power or 0.5) * 300
sensitivity = clamp(float(buyer.price_sensitivity or 0.5), 0.0, 1.0)

# 有折扣时用折后价，否则用原价
effective_price = float(discount_item.final_price) if discount_item else float(listing.price or 0)
effective_price = max(1, effective_price)

price_gap = (effective_price - target_price) / max(target_price, 1)

if price_gap >= 0:
    # 高于目标价：越贵分越低，price_sensitivity 越高惩罚越重
    price_score = clamp(1 - price_gap * (0.5 + sensitivity * 0.5), 0.0, 1.0)
else:
    # 低于目标价：越便宜分越高，price_sensitivity 越低奖励越大
    price_score = clamp(1 + abs(price_gap) * (0.5 - sensitivity * 0.5), 0.0, 1.0)
```

### 3.2 price_sensitivity 的语义

| sensitivity 值 | 含义 | 高于目标价时 | 低于目标价时 |
|----------------|------|-------------|-------------|
| 0.0 | 完全不在乎价格 | 惩罚系数 0.5（最轻） | 奖励系数 0.5（最大） |
| 0.5（默认） | 中等价格敏感 | 惩罚系数 0.75 | 奖励系数 0.25 |
| 1.0 | 极度价格敏感 | 惩罚系数 1.0（最重） | 奖励系数 0.0（无额外奖励） |

一句话：**价格敏感度越高，对高价越排斥，但对低价的额外奖励越小（因为已经满分了）。**

---

## 4. 折扣的传导路径

```
折扣活动 ongoing
    ↓
effective_price = final_price（折后价）
    ↓
price_gap 缩小（折后价更接近或低于目标价）
    ↓
price_score 提升
    ↓
score 提升
    ↓
order_prob = clamp(0.08 + score * 0.95, 0.05, 0.90) 提升
```

折扣通过降低成交价，使价格更贴近买家心理价位，从而提升下单概率。

---

## 5. 效果示例

假设买家 `purchase_power=0.5`（目标价 180 RM），`price_sensitivity=0.5`：

| 商品原价 | 折扣 | 折后价 | price_gap | price_score |
|----------|------|--------|-----------|-------------|
| 200 RM | 无 | 200 RM | +0.11 | 0.917 |
| 200 RM | 20% | 160 RM | -0.11 | 1.0（clamp） |
| 300 RM | 无 | 300 RM | +0.67 | 0.498 |
| 300 RM | 30% | 210 RM | +0.17 | 0.873 |
| 300 RM | 50% | 150 RM | -0.17 | 1.0（clamp） |

## 6. 与现有权重体系的关系

`price_score` 权重在总评分中保持 **0.22** 不变，无需调整其他维度权重。折扣的影响完全通过改变 `price_score` 的输入值（`effective_price`）传导，不新增维度。

```python
score = (
    0.32 * category_match
    + 0.22 * price_score      # 折扣通过此项传导
    + 0.14 * quality_score
    + 0.08 * stock_score
    + 0.16 * base_intent
    + 0.08 * impulse
)
```

---

## 7. 验收标准

1. 折扣生效时，`effective_price` 使用 `final_price` 而非 `listing.price`
2. 高于目标价时，`price_sensitivity` 越高惩罚越重
3. 低于目标价时，`price_score` 不低于无折扣时的值
4. `price_sensitivity=0` 的买家，折扣对其 `price_score` 奖励最大
5. 无折扣时行为与改造前一致（`sensitivity=0.5` 时结果接近原公式）
