# 56-Shopee 买家中心展示上架商品前后端数据库 Redis 实现设计

> 创建日期：2026-05-14  
> 状态：V1 已实现

## 1. 目标

基于当前已完成的 Shopee 买家中心前端界面，将页面中的静态商品卡片替换为当前对局中用户已经上架的真实商品列表，让学生可以从买家视角查看自己店铺对外展示的商品。

本设计只定义数据接入与业务口径，不修改现有买家中心页面样式和布局：

- 页面文件：`frontend/src/modules/shopee/views/BuyerCentreView.tsx`
- 入口页面：`frontend/src/modules/shopee/ShopeePage.tsx`
- 后端接口文件：`backend/apps/api-gateway/app/api/routes/shopee.py`
- 现有商品数据来源：`shopee_listings`、`shopee_listing_variants`、商品图片与仓库关联数据

## 2. 范围与非范围

### 2.1 本期范围

- 买家中心 `/shopee/buyer-centre` 保持当前橙色顶部、搜索框、类目 Tab、5 列商品卡、分页和返回卖家中心布局。
- 商品列表从当前 `mockProducts` 切换为后端接口返回的当前 run/user 已上架主商品。
- 仅展示 `status=live` 的对外可售主商品。
- 买家中心首页按主商品维度展示，不展开 SKU/变体，不使用变体名称、变体图片或变体规格作为卡片主体信息。
- 商品卡片展示主商品主图、主商品标题、主商品价格口径、主商品销量、评分/评价数占位和商城标签占位。
- 支持关键字搜索、类目筛选和分页查询。
- 历史对局回溯模式下允许只读浏览商品列表，不允许购买、加购或咨询等写操作。
- 接入 Redis 短 TTL 列表缓存，并在商品发布、编辑、下架、库存/销量变化后失效。

### 2.2 非范围

- 不重做买家中心 UI，不调整颜色、宽度、卡片网格、顶部区域或分页样式。
- 不实现购物车、下单、收藏、关注、买家登录态、店铺主页、商品详情页。
- 不接入真实 Shopee 平台买家端 API。
- 不新增评价系统、评分系统或真实评论表；评分和评价数本期可由后端返回默认占位。
- 不展示其他玩家或全平台商品，本期只展示当前对局当前用户自己的上架商品。
- 不改变订单模拟、营销活动或客服触发规则。

## 3. 当前前端界面观察

当前 `BuyerCentreView.tsx` 已完成买家端静态页面结构：

| 区块 | 当前内容 | 本期保留方式 |
| --- | --- | --- |
| 顶部橙色导航 | Shopee 风格顶部、返回卖家中心按钮 | 保留布局，仅保留返回行为 |
| 搜索框 | 静态输入框，按钮无真实查询 | 接入 keyword 查询，不改视觉 |
| 类目 Tab | `allCategories` 静态数组，前 6 个展示，其余下拉 | 优先用接口返回类目；无数据时保留前端默认类目 |
| 商品标题 | `全部商品` | 保留 |
| 商品网格 | `mockProducts` 静态 12 条，5 列卡片 | 替换为接口返回 items |
| 商品主图 | 灰色占位文字 | 接口有 `cover_url` 时展示图片；无图时保留占位 |
| 分页器 | 静态 1/2 页和跳转输入 | 接入后端 `page/page_size/total/total_pages` |

现有页面为独立全屏 `fixed inset-0` 买家端页面，不渲染卖家中心顶部栏、左侧栏、右侧栏、通知抽屉或客服抽屉；本期继续保持该边界。

## 4. 页面展示规则

### 4.1 商品可见范围

买家中心只展示满足以下条件的主商品：

- `run_id` 等于当前对局。
- `user_id` 等于当前登录用户。
- 主商品状态为 `live`。
- 主商品存在可售库存，或其下属变体聚合后存在可售库存；但首页展示仍使用主商品信息。
- 商品未被后台质量或发布校验标记为不可展示状态。

首页商品卡片的主体口径必须固定为主商品：

- 图片读取 `shopee_listings.cover_url` 或主商品图片表中的封面图，不读取 `shopee_listing_variants.image_url`。
- 标题读取 `shopee_listings.title`，不拼接变体名、规格名或选项值。
- 价格读取主商品价格字段，或在主商品层返回变体价格聚合后的展示价格；前端不直接展示某个具体变体价格。
- 销量读取主商品 `sales_count`，或由后端聚合后写回/返回主商品销量口径；前端不展示单个变体销量。

推荐后端按以下顺序过滤：

1. run/user 权限过滤。
2. listing 状态过滤。
3. 变体/库存可售性过滤。
4. 搜索、类目、分页。

### 4.2 商品卡片字段

| 前端展示 | 后端字段 | 说明 |
| --- | --- | --- |
| TOP 排名 | `rank` | 按排序结果计算，当前页从 `(page-1)*page_size+1` 递增 |
| 商城标签 | `is_mall` | 本期默认 `false`，后续如有店铺认证再打开 |
| 商品主图 | `cover_url` | 只读取主商品主图；无主商品图则前端占位，不回退变体图 |
| 商品标题 | `title` | 读取主商品标题，不拼接变体信息 |
| 价格 | `display_price` | 由后端按主商品口径返回展示价格；可使用主商品价或主商品层聚合价格区间，前端不直接选取某个变体价格 |
| 已售 | `sales_count` | 主商品销量口径，不展示单个变体销量 |
| 评分星级 | `rating` | 本期默认 `5.0` 或后端返回占位值 |
| 评价数 | `review_count` | 本期默认 `0`，不新增评价表 |
| 商品 ID | `listing_id` | 用于后续商品详情页或调试，不直接显示也可返回 |

### 4.3 搜索规则

搜索框输入关键字后点击搜索按钮：

- 请求参数 `keyword` 支持商品标题、SKU 编码、商品 ID 的模糊匹配。
- 搜索时重置到第 1 页。
- 空关键字表示查询全部商品。
- 搜索只影响买家中心商品列表，不影响卖家中心“我的产品”页面筛选状态。

### 4.4 类目规则

类目 Tab 用于筛选当前用户已上架商品的商品类目：

- 推荐接口返回 `categories`，基于当前 run/user 的 live listing 聚合生成。
- 默认 Tab 可为 `全部商品` 或继续保留当前页面的首个类目高亮；实现时如要最小改动，可保留页面标题为“全部商品”，Tab 选中态独立筛选。
- 点击类目时请求 `category={category_name}` 并重置到第 1 页。
- 下拉中的类目与顶部前 6 个类目来自同一份 `categories`。
- 若当前对局没有上架商品或没有类目数据，保留当前静态类目作为页面占位，不发起错误提示。

### 4.5 排序规则

本期默认排序：

1. 销量 `sales_count` 降序。
2. 上架/更新时间倒序。
3. listing ID 倒序。

后续如需买家端“综合 / 最新 / 销量 / 价格”排序，可单独设计筛选栏；本期不新增排序 UI。

## 5. 后端接口设计

接口文件沿用：`backend/apps/api-gateway/app/api/routes/shopee.py`。

### 5.1 买家中心商品列表

```http
GET /shopee/runs/{run_id}/buyer-centre/products
```

查询参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `keyword` | string | 空 | 商品标题、SKU、商品 ID 模糊搜索 |
| `category` | string | 空 | 类目筛选；空表示全部 |
| `page` | int | 1 | 页码，从 1 开始 |
| `page_size` | int | 10 | 每页数量；前端 5 列布局建议使用 10 或 15 |

响应示例：

```json
{
  "meta": {
    "run_id": 6,
    "user_id": 3,
    "read_only": false,
    "currency": "RM",
    "keyword": "",
    "category": "",
    "page": 1,
    "page_size": 10,
    "total": 12,
    "total_pages": 2
  },
  "categories": [
    { "name": "美妆个护", "count": 8 },
    { "name": "家居生活", "count": 4 }
  ],
  "items": [
    {
      "listing_id": 101,
      "rank": 1,
      "is_mall": false,
      "title": "便携式补水喷雾",
      "category": "美妆个护",
      "cover_url": "https://example.test/product.jpg",
      "display_price": "18.00",
      "min_price": 18.0,
      "max_price": 22.0,
      "sales_count": 36,
      "stock_available": 120,
      "rating": 5.0,
      "review_count": 0,
      "status": "live"
    }
  ]
}
```

### 5.2 后端校验

- 当前用户必须有权限访问该 `run_id`。
- `page` 小于 1 时按 1 处理或返回校验错误；建议前端传入前先规整。
- `page_size` 建议限制在 `1~50`，避免买家中心一次性加载过多图片。
- 历史 finished run 允许只读查询。
- 查询接口不得触发订单模拟、客服触发、商品状态变更或库存变更。

## 6. 数据模型设计

本期优先复用现有上架商品相关表，不新增数据库表：

- `shopee_listings`
- `shopee_listing_variants`
- 商品图片/媒体相关现有表或 JSON 字段
- 仓库商品关联字段 `product_id`

字段来源建议：

| 返回字段 | 数据来源 |
| --- | --- |
| `listing_id` | `shopee_listings.id` |
| `title` | `shopee_listings.title` |
| `category` | listing 保存的类目字段，或关联选品商品类目 |
| `cover_url` | listing 主图或主商品图片表封面图；禁止使用变体图兜底 |
| `min_price/max_price` | 后端可用变体价格聚合生成主商品展示价格区间；前端仍按主商品一张卡展示 |
| `sales_count` | listing 主商品销量字段；如需聚合变体销量，应由后端聚合为主商品销量口径后返回 |
| `stock_available` | 主商品可售库存字段，或后端聚合变体库存后作为主商品库存口径返回 |
| `status` | `shopee_listings.status` |

如果实现时发现现有表缺少买家中心必须展示的字段，不直接新增大表；优先从已存在的商品发布、变体、图片和库存字段聚合生成。

## 7. Redis 缓存设计

买家中心商品列表属于读多写少页面，建议接入短 TTL 缓存。

| 场景 | 建议 key | TTL | 失效时机 |
| --- | --- | --- | --- |
| 商品列表 | `shopee:buyer_centre:products:{run_id}:{user_id}:{query_hash}` | 30 秒 | 商品发布、编辑、下架、库存/销量变化 |
| 类目聚合 | 可并入列表响应 | 30 秒 | 同列表 |

失效规则：

- 新商品发布为 `live` 后清理买家中心列表缓存。
- 商品编辑保存后清理买家中心列表缓存。
- 商品下架或状态变更后清理买家中心列表缓存。
- 订单生成导致销量和库存变化后清理买家中心列表缓存。
- 取消订单、发货、补货、库存同步等影响可售库存时，若当前买家中心展示库存或售罄状态，也应清理缓存。

缓存失败不得阻断接口读取；Redis 不可用时直接走数据库查询。

## 8. 前端实现设计

### 8.1 Props 与数据加载

`BuyerCentreView` 建议补充必要 props：

- `runId: number | null`
- `readOnly?: boolean`
- `onBackToSellerCentre: () => void`

页面加载时：

1. 根据 `runId` 请求商品列表。
2. 请求成功后替换 `mockProducts`。
3. 请求失败时展示轻量错误提示和重试入口。
4. 请求中显示商品网格区域 loading 状态，不改变页面整体布局。

### 8.2 保留当前布局

实现时只替换数据来源与事件处理：

- 不改变 `fixed inset-0 bg-[#f5f5f5]` 外层结构。
- 不改变顶部橙色 Header、高度、Logo、搜索框视觉。
- 不改变 `max-w-[1200px]` 主容器。
- 不改变 `grid grid-cols-5 gap-2.5` 商品卡片网格。
- 不改变商品卡片 hover、边框、TOP 角标和星级区域视觉。
- 不新增卖家中心左侧栏、右侧栏或顶部面包屑。

### 8.3 空状态

当接口返回 `items=[]`：

- 商品网格区域展示空状态文案：`暂无上架商品`。
- 若存在搜索或类目筛选，展示：`没有找到符合条件的商品`。
- 保留顶部搜索、类目 Tab 和返回卖家中心按钮。

### 8.4 分页

分页器行为：

- 页码按钮根据 `total_pages` 动态生成，当前页高亮沿用橙色。
- 上一页/下一页根据边界禁用。
- 输入页码点击 `Go` 后跳转到目标页，越界时规整到有效范围。
- 搜索和切换类目时重置到第 1 页。

## 9. 与现有业务的关系

### 9.1 与我的产品

买家中心读取的是“我的产品”发布后的对外视图：

- “我的产品”负责卖家编辑、库存、状态、批量操作。
- “买家中心”只负责买家视角展示，不提供卖家管理操作。
- 商品状态从 `live` 变为下架/草稿后，应从买家中心消失。

### 9.2 与订单模拟

本期不改变订单模拟逻辑：

- 买家中心不会触发订单模拟。
- 点击商品卡片本期可不做跳转，或仅作为后续商品详情页入口预留。
- 订单模拟产生销量和库存变化后，买家中心下一次刷新应看到最新销量/库存口径。

### 9.3 与客服系统

本期不从买家中心直接创建客服会话：

- 售前商品细节追问仍由订单模拟链路按规则触发。
- 买家中心上的搜索、浏览和商品卡片点击不生成客服会话。
- 后续若需要“联系卖家”按钮，应单独设计买家主动咨询触发规则。

## 10. 验收标准

实现完成后应满足：

1. `/shopee/buyer-centre` 保持当前页面样式、布局、宽度、卡片网格和返回卖家中心行为不变。
2. 页面不再展示 `mockProducts`，而是展示当前 run/user 已上架 `live` 商品。
3. 没有上架商品时展示空状态，不报错、不跳回卖家中心。
4. 商品卡片正确展示主图、标题、价格、销量、评分/评价数占位和 TOP 排名。
5. 搜索商品标题后，只展示匹配商品，并重置到第 1 页。
6. 点击类目 Tab 后，只展示该类目商品，并重置到第 1 页。
7. 分页器可切换页码、上一页/下一页和 Go 跳转。
8. 历史对局回溯模式可只读查看商品列表，不触发任何写入。
9. 商品发布、编辑、下架、订单销量变化后，买家中心缓存会失效或在短 TTL 后刷新。
10. 后端接口只查询当前 run/user 商品，不泄露其他用户或其他对局商品。

## 11. 建议验证用例

| 用例 | 操作 | 预期 |
| --- | --- | --- |
| 无商品 | 新对局未发布商品时进入买家中心 | 展示空状态 |
| 单页商品 | 发布 3 个 live 商品后进入买家中心 | 5 列网格中展示 3 张商品卡 |
| 多页商品 | 发布超过 page_size 的商品 | 分页器可切换，排名连续 |
| 搜索 | 输入商品标题关键词搜索 | 只返回匹配商品 |
| 类目筛选 | 点击某类目 Tab | 只返回该类目 live 商品 |
| 下架商品 | 将商品从 live 改为下架 | 买家中心刷新后不再显示该商品 |
| 销量变化 | 订单模拟生成该商品订单 | 商品卡片已售数刷新 |
| 历史对局 | finished run 进入买家中心 | 可只读浏览，无写操作触发 |
