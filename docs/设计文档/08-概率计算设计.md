# 08-概率计算设计

## 1. 目标
为“商品上架后订单模拟”提供一套可解释、可调参、可扩展的概率模型。  
本版采用 **非连乘主模型**，避免多项小概率连乘导致结果过小。

## 2. 总体思路
不再直接使用多个小概率项连乘。  
改为：
1. 先计算综合评分 `S`
2. 再用 Sigmoid 将评分映射到下单概率 `P_order`
3. 用活跃买家规模计算期望订单，再做随机采样

## 3. 核心公式

### 3.1 单商品下单概率
\[
S = b + w_1\cdot 商品分 + w_2\cdot 人群匹配分 + w_3\cdot 时段分 + w_4\cdot 运营分 + w_5\cdot 竞争项 + \epsilon
\]

\[
P_{order}=\sigma(S)=\frac{1}{1+e^{-S}}
\]

- `b`：基础偏置（控制全局难易度）
- `epsilon`：随机扰动（建议 \[-0.1, 0.1\]）
- 其余分项建议归一化到 `0~1`（竞争项可为负）

### 3.2 时段订单数
\[
Orders_t \sim Poisson(ActiveUsers_t \times P_{order})
\]

- `ActiveUsers_t`：该时段活跃买家数
- `P_order`：该时段该商品平均下单概率

## 4. 分项定义（MVP）

### 4.1 商品分（0~1）
由以下维度加权：
- 标题质量
- 类目准确度
- 图片/视频完整度
- 价格竞争力
- 库存健康度

示例：
\[
商品分 = 0.20\cdot标题 + 0.20\cdot类目 + 0.20\cdot素材 + 0.20\cdot价格 + 0.20\cdot库存
\]

### 4.2 人群匹配分（0~1）
由人物画像与商品匹配得到：
- 类目偏好匹配
- 品牌偏好匹配
- 价格带匹配

示例：
\[
人群匹配分 = 0.40\cdot类目匹配 + 0.30\cdot品牌匹配 + 0.30\cdot价格匹配
\]

### 4.3 时段分（0~1）
由时段流量与人群活跃偏好决定：
- 小时因子（24小时）
- 星期因子（7天）
- 节假日因子（可选）

可先算成指数后再归一到 `0~1`。

### 4.4 运营分（0~1）
用于防止长期0单：
- 保底曝光策略
- 冷启动保护（上架前48小时）
- 0单回补（按天小幅上调点击/转化）

### 4.5 竞争项（建议取负值）
单机：固定 `0`。  
多人：使用相对强度惩罚：
\[
竞争项 = -\ln\left(1+\frac{Strength_{opp}}{Strength_{self}}\right)
\]

并建议裁剪到：
\[
竞争项 \in [-1.2, 0]
\]

## 5. 强度（Strength）计算
多人模式下，自己和对手都用同一套强度公式：

\[
Strength = 100\times(0.30\cdot价格竞争力 + 0.25\cdot内容质量 + 0.20\cdot履约能力 + 0.15\cdot店铺信号 + 0.10\cdot转化信号)
\]

其中各子项均归一化到 `0~1`：
- 价格竞争力：同类价格相对优势
- 内容质量：素材与信息完整度
- 履约能力：库存、时效、运费
- 店铺信号：店铺稳定性/评级（MVP 可默认中位值）
- 转化信号：历史CTR/CVR（MVP 可默认中位值）

多个对手时：
\[
Strength_{opp}=\sum_j \alpha_j \cdot Strength_j
\]
`α_j` 可按相似度加权（MVP 可先均分）。

## 6. 默认参数建议（MVP）
建议初始参数：
\[
S = -2.2 + 1.2\cdot商品分 + 1.0\cdot人群匹配分 + 0.8\cdot时段分 + 0.6\cdot运营分 + 0.9\cdot竞争项
\]

- 单机：`竞争项=0`
- `epsilon`：每次采样注入小噪声（如 `U(-0.1, 0.1)`）

## 7. 示例
假设某商品在某时段：
- 商品分 = 0.72
- 人群匹配分 = 0.65
- 时段分 = 0.55
- 运营分 = 0.60
- 单机竞争项 = 0
- 扰动 `epsilon = 0.03`

则：
\[
S=-2.2 + 1.2*0.72 + 1.0*0.65 + 0.8*0.55 + 0.6*0.60 + 0 + 0.03 = 0.144
\]
\[
P_{order}=\sigma(0.144)\approx 0.536
\]

若本时段 `ActiveUsers_t = 20`：
\[
\lambda = 20 * 0.536 = 10.72,\quad Orders_t \sim Poisson(10.72)
\]

即该时段大约会产生 8~13 单（随机波动）。

## 8. 与现有系统结合方式
1. 保留现有“买家池 + 时段活跃”逻辑。  
2. 将原“多概率连乘”替换为本文件 `S -> Sigmoid` 方案。  
3. 防0单策略作为 `运营分` 的输入，不单独连乘。  
4. 单机先固定 `竞争项=0`，多人再启用强度竞争。

## 9. 验收标准
- 不出现长期“全0单”且不过度爆单。
- 晚高峰订单显著高于低谷时段。
- 类目/价格匹配度变化能显著影响下单率。
- 单机与多人模式切换时，仅竞争项逻辑变化。
