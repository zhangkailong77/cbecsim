# 07-商品发布第二步设计

## 1. 目标
- 在“添加新商品”第一步填写基础信息并点击“下一步”后，进入第二步完善页。
- 第二步页面沿用 Shopee 工作台现有蓝白视觉、字号与间距体系，文案统一中文。
- 上一步填写的关键信息（1:1/3:4 图片、商品名、类目、GTIN）在第二步对应表单中自动回填显示。
- 通过后端草稿机制将关键数据先落库，再由“保存并发布/保存并下架”生成正式商品，满足状态持久化要求。
- 选择类目后自动展示该类目规格字段（动态属性），并持久化到数据库。

## 2. 流程
1. 用户在第一步录入商品名称、类目、GTIN，并上传 1:1（必填）与可选 3:4 图片。
2. 点击“下一步”后，前端调用创建草稿接口：
   - 图片上传到 OSS。
   - 草稿主表与草稿图片表写入数据库。
3. 前端进入第二步页面，显示“填写建议 + 基本信息 + 商品描述 + 底部操作栏”。
4. 用户在第二步可继续编辑商品名、类目、GTIN、描述。
5. 用户在第二步可继续追加 1:1 图片、3:4 图片，并可上传/替换商品视频（MP4）。
6. 点击“保存并发布”或“保存并下架”时：
   - 先调用草稿更新接口保存最新字段。
   - 再调用草稿发布接口生成正式商品（状态为 `live` 或 `unpublished`）。
7. 选择类目后，前端调用规格模板接口并渲染动态规格输入区，规格值随草稿保存。

## 3. 规则
- 商品图片上传统一走 OSS，禁止写本地兜底。
- 草稿创建至少要求 1 张 1:1 图片。
- 1:1 与 3:4 图片各自最多 9 张。
- 商品标题长度要求 2~120 字符。
- 草稿仅允许所属用户在所属 run 下访问和发布。
- 发布后草稿状态更新为 `published`，正式商品进入 `shopee_listings`。

## 4. 数据模型
### 4.1 新增表 `shopee_listing_drafts`
- `id` 主键
- `run_id` 对局 ID
- `user_id` 用户 ID
- `category_id` 三级类目节点 ID（关联 `shopee_category_nodes.id`）
- `title` 商品标题
- `category` 类目路径（如 `美妆个护 > 彩妆 > 底妆`）
- `gtin` GTIN
- `description` 商品描述
- `cover_url` 封面图 URL
- `status` 草稿状态（默认 `drafting`）
- `created_at` / `updated_at`

### 4.2 新增表 `shopee_listing_draft_images`
- `id` 主键
- `draft_id` 草稿 ID
- `image_url` 图片 URL
- `image_ratio` 比例（`1:1` / `3:4`）
- `sort_order` 排序
- `is_cover` 是否封面
- `created_at`

### 4.3 扩展表 `shopee_listings`
- 新增字段 `gtin`
- 新增字段 `description`

### 4.4 新增表 `shopee_category_nodes`（类目树）
- `id` 主键
- `parent_id` 父类目节点（根类目为 `NULL`）
- `level` 层级（1/2/3）
- `name` 节点名
- `path` 完整路径（唯一）
- `sort_order` 排序
- `is_active` 是否启用

### 4.5 新增表 `shopee_spec_templates`（类目规格字段定义）
- `id` 主键
- `category_id` 对应三级类目节点
- `attr_key` 字段键（前端可作为“字段1/字段2”的唯一标识）
- `attr_label` 字段名（展示文案）
- `input_type` 字段类型（`text` / `select`）
- `is_required` 是否必填
- `sort_order` 排序
- `is_active` 是否启用

### 4.6 新增表 `shopee_spec_template_options`（下拉选项）
- `id` 主键
- `template_id` 关联 `shopee_spec_templates.id`
- `option_value` 选项值
- `option_label` 选项展示名
- `sort_order` 排序
- `is_active` 是否启用

### 4.7 新增表 `shopee_listing_draft_spec_values`
- `draft_id` 草稿 ID
- `attr_key` 属性键
- `attr_label` 属性名
- `attr_value` 属性值

### 4.8 新增表 `shopee_listing_spec_values`
- `listing_id` 正式商品 ID
- `attr_key` 属性键
- `attr_label` 属性名
- `attr_value` 属性值

## 5. 接口
- `POST /shopee/runs/{run_id}/product-drafts`
  - 用途：创建商品草稿（含图片上传 OSS）
- `GET /shopee/categories/tree`
  - 用途：获取三级类目树（来自数据库配置）
- `GET /shopee/spec-templates?category_id=...`
  - 用途：按三级类目返回动态规格模板
- `GET /shopee/runs/{run_id}/product-drafts/{draft_id}`
  - 用途：读取草稿详情
- `PUT /shopee/runs/{run_id}/product-drafts/{draft_id}`
  - 用途：更新草稿基础字段（标题/类目/GTIN/描述）及规格值
- `POST /shopee/runs/{run_id}/product-drafts/{draft_id}/assets`
  - 用途：第二步追加素材（1:1、3:4、视频）
- `POST /shopee/runs/{run_id}/product-drafts/{draft_id}/publish`
  - 用途：发布草稿生成正式商品
  - `status_value=live`：保存并发布
  - `status_value=unpublished`：保存并下架

## 6. 验收标准
- 从“添加新商品”点击“下一步”后进入第二步页面，且样式与现有页面一致。
- 第二步页面能正确显示第一步已填写信息。
- 选择类目后应展示该类目的规格输入区，且类目不同字段不同。
- 第二步底部固定操作栏包含：取消、保存并下架、保存并发布。
- 点击“保存并发布/保存并下架”后，数据库可查到正式商品与图片数据，状态正确。
- 规格字段值可在草稿表与正式商品表中查询到。
- 所有图片 URL 来源为 OSS，且草稿/正式商品均非仅前端本地状态。
